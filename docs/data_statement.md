# Data Statement

## Dataset role

The included CSV files are used for educational binary text classification. They are suitable for learning how to train, evaluate, and deploy a simple NLP model, but they are not sufficient for real-world misinformation detection.

## Files

- `data/True.csv`: real-news examples
- `data/Fake.csv`: fake-news examples

The current training script uses the `text` column by default and prepends `title` only when a title column exists.

## Current dataset profile

Generated by `python src/train_model.py`:

- Total rows before deduplication: 1,998
- Rows after deduplication: 1,991
- Duplicate rows removed: 7
- REAL rows after deduplication: 992
- FAKE rows after deduplication: 999

## Known leakage and bias

The dataset contains strong source/style artifacts:

- Nearly all REAL examples contain Reuters-style text.
- Almost no FAKE examples contain Reuters-style text.
- REAL and FAKE labels are strongly associated with different `subject` values.

As a result, high accuracy may reflect source recognition rather than factuality detection.

## Recommended improvements

Before using this project beyond a demo:

1. Source-balance the dataset.
2. Remove boilerplate/source prefixes where appropriate.
3. Deduplicate across labels and sources.
4. Create a time-based split.
5. Validate on external datasets from different publishers and years.
6. Report performance separately by source, topic, and date range.
